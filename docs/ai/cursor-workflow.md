# Cursor Workflow

## Atajos de teclado más usados
- **Ctrl + `** → Abrir terminal
- **Ctrl + K** → Edición inline, para pedir explicaciones o modificar código
- **Ctrl + L** → Chat contextual
- **Ctrl + Shift + I** → Composer, para cambios en varios archivos
- **Tab** → Aceptar sugerencia de autocompletado

** Ejemplos donde cursor mejora mi código

### Ejemplo 1 - Autocompletado
Al escribir un comentario describiendo una función, Cursor generó
automáticamente el código completo adaptado a mi proyecto.

### Ejemplo 2 — Explicación de código
Usando Ctrl + K pude pedir a Cursor que explicara paso a paso
la función renderizarTareas sin tener que entenderla línea por línea.

### Ejemplo 3 — Edición inline
Usando Ctrl + K pedí a Cursor que añadiera una confirmación 
antes de eliminar una tarea. Generó automáticamente el código 
correcto integrándolo en la función existente.

### Ejemplo 4 — Composer
Usando Ctrl + Shift + I pedí a Cursor que añadiera un mensaje 
cuando no hay tareas en la lista. Modificó automáticamente 
la función renderizarTareas añadiendo la comprobación correcta.
